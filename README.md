# file-pin-cli

This is a command-line interface (CLI) application for directly controlling an instance of [ipfs-file-pin-service](https://github.com/Permissionless-Software-Foundation/ipfs-file-pin-service).

This CLI was forked from [psf-ipfs](https://github.com/Permissionless-Software-Foundation/psf-ipfs). This CLI has all the same commands as that one, plus the addition of IPFS-specific commands.

## Installation

This software requires node.js v20 or higher. Instructions for installation:

- `git clone https://github.com/Permissionless-Software-Foundation/file-pin-cli`
- `cd file-pin-cli`
- `npm install`
- `cp .env.example .env`

Customize the .env file to suite your needs. The defaults should work.


## Usage

### Display Help

- `node file-pin.js help`

### Wallet Commands

#### Create Wallet

Create a new BCH wallet:

- `node file-pin.js wallet-create -n myWallet -d "My test wallet"`

##### Arguments
- Use `-n` to specify the wallet name (required).
- Use `-d` to specify a description for the wallet (optional).


#### List Wallets

List all existing wallets:

- `node file-pin.js wallet-list`

##### Arguments
- none


#### Wallet Addresses

List the different addresses (cash, SLP, legacy) for a wallet:

- `node file-pin.js wallet-addrs -n myWallet`

##### Arguments
- Use `-n` to specify the wallet name (required).


#### Wallet Balance

Get the BCH and SLP token balances held by a wallet:

- `node file-pin.js wallet-balance -n myWallet`

##### Arguments
- Use `-n` to specify the wallet name (required).


#### Wallet Sweep

Sweep BCH and SLP tokens from a WIF private key into a wallet:

- `node file-pin.js wallet-sweep -n myWallet -w L1tcMMxABb...`

##### Arguments
- Use `-n` to specify the wallet name receiving the funds (required).
- Use `-w` to specify the WIF private key to sweep (required).


#### Send BCH

Send BCH to an address:

- `node file-pin.js send-bch -n myWallet -a bitcoincash:qqs2wrahl6azn9qdyrmp9ygeejqvzr8ruv7e9m30fr -q 0.001`

##### Arguments
- Use `-n` to specify the wallet name sending BCH (required).
- Use `-a` to specify the address to send BCH to (required).
- Use `-q` to specify the quantity of BCH to send (required).


#### Send Tokens

Send SLP tokens to an address:

- `node file-pin.js send-tokens -n myWallet -a bitcoincash:qqs2wrahl6azn9qdyrmp9ygeejqvzr8ruv7e9m30fr -q 1 -t 38e97c5d7d3585a2cbf3f9580c82ca33985f9cb0845d4dcce220cb709f9538b0`

##### Arguments
- Use `-n` to specify the wallet name sending the tokens (required).
- Use `-a` to specify the address to send tokens to (required).
- Use `-q` to specify the quantity of tokens to send (required).
- Use `-t` to specify the token ID of the token to send (required).


### Message Commands

#### Sign Message

Cryptographically sign a message using the wallet's private key:

- `node file-pin.js msg-sign -n myWallet -m "Hello world"`

##### Arguments
- Use `-n` to specify the wallet name to sign the message with (required).
- Use `-m` to specify the message to sign (required).


#### Verify Message

Verify that a message signature was generated by a specific BCH address:

- `node file-pin.js msg-verify -a bitcoincash:qqs2wrahl6azn9qdyrmp9ygeejqvzr8ruv7e9m30fr -m "Hello world" -s HxH6yR...`

##### Arguments
- Use `-a` to specify the BCH address that signed the message (required).
- Use `-m` to specify the cleartext message that was signed (required).
- Use `-s` to specify the signature to verify (required).


### IPFS Commands


#### Get Node Status

Get information about the node and the other nodes it is connected to:

- `node file-pin.js ipfs-status`

##### Arguments
- none


#### IPFS Peers

Get information about the peers connected to the node:

- `node file-pin.js ipfs-peers`

##### Arguments
- Use the `-a` flag to display verbose information about the peers (optional).


#### IPFS Relays

Get information about the [V2 Circuit Relays](https://cashstack.info/docs/local-back-end/circuit-relay) relays connected to the node:

- `node file-pin.js ipfs-relays`

##### Arguments
- none


#### IPFS Connect

Instruct the node to attempt to connect to a specific peer:

- `node file-pin.js ipfs-connect -d -m /ip4/143.198.70.59/tcp/4001/p2p/12D3KooWMbU9R49aiYUeFBpxFYK6PggacoeMydaZaR2dzDpWgcA6`

##### Arguments
- Use the `-m` flag to specify the multiaddr of the peer to connect to (required).
- Use the `-d` flag to display debug information (optional).


#### Wallet Service

Query the instance of [ipfs-bch-wallet-consumer](https://cashstack.info/docs/local-back-end/ipfs-bch-wallet-consumer) to retrieve the instances of [ipfs-bch-wallet-service](https://cashstack.info/docs/global-back-end/ipfs-bch-wallet-service) (blockchain services) and [ipfs-file-pin-service](https://cashstack.info/docs/global-back-end/file-storage#ipfs-file-pin-service) (file storage services) it is connected to.

- `node file-pin.js wallet-service`

##### Arguments
- none


### File Commands


#### Download File

Download a file from IPFS to the `files/` directory. This looks up the filename from the pin status metadata using the CID.

- `node file-pin.js download-file -c bafybeicntn33k36dtrqd6ssf45duek5m6wz55antcdt4k4ltmq2u3sj2jm`

##### Arguments
- Use `-c` to specify the [CID](https://docs.ipfs.tech/concepts/content-addressing/) of the file (required).


#### Download CID

Download a file from IPFS to the `files/` directory by specifying both the CID and a filename. This downloads a raw CID without requiring a valid Pin Claim.

- `node file-pin.js download-cid -c bafybeicntn33k36dtrqd6ssf45duek5m6wz55antcdt4k4ltmq2u3sj2jm -f myfile.json`

##### Arguments
- Use `-c` to specify the [CID](https://docs.ipfs.tech/concepts/content-addressing/) of the file (required).
- Use `-f` to specify the filename to save the file as (required).


### Pin Commands


#### Pin Status

Get pin status information about a file identified by its CID:

- `node file-pin.js pin-status -c bafybeicntn33k36dtrqd6ssf45duek5m6wz55antcdt4k4ltmq2u3sj2jm`

##### Arguments
- Use `-c` to specify the [CID](https://docs.ipfs.tech/concepts/content-addressing/) of the file (required).


#### Pin Upload

Upload a file from the `files/` directory and pin it to IPFS:

- `node file-pin.js pin-upload -f mutable-67ccefcca67097473e78ca10.json`

##### Arguments
- Use `-f` to specify the file name in the `files/` directory (required).


#### Pin Claim

Submit a pin claim to the pinning service for a given CID. This requires a Proof-of-Burn and Claim transaction to have already been generated on the BCH blockchain.

- `node file-pin.js pin-claim -p be4b63156c93f58ed311d403d9f756deda9abbc81d0fef8fbe5d769538b4261c -t c71e2f2cdf8658d90c61ac6183b8ffeeb359779807b317386044705d8352f0f2 -f mutable-67ccefcca67097473e78ca10.json -a bitcoincash:qqs2wrahl6azn9qdyrmp9ygeejqvzr8ruv7e9m30fr -c bafybeied3zdwdiro7fqytyha2yfband4lwcrtozmf6shynylt3kexh26dq`

##### Arguments
- Use `-p` to specify the Proof of Burn transaction ID (required).
- Use `-t` to specify the Claim transaction ID (required).
- Use `-f` to specify the file name (required).
- Use `-a` to specify the address to claim the pin to (required).
- Use `-c` to specify the CID of the file (required).


#### Pin Claim File

Upload a file to IPFS and generate a Pin Claim on the BCH blockchain in a single step. This combines the `pin-upload`, on-chain Proof-of-Burn and Pin Claim transaction generation, and pinning service notification into one command.

- `node file-pin.js pin-claim-file -f myfile.json -n myWallet`

##### Arguments
- Use `-f` to specify the file name in the `files/` directory (required).
- Use `-n` to specify the wallet name to pay for the pin claim (required).


#### Pin Renew

Renew an expired (or soon-to-expire) pin claim for a CID. This generates a new Proof-of-Burn and Pin Claim transaction on the BCH blockchain.

- `node file-pin.js pin-renew -c bafybeicntn33k36dtrqd6ssf45duek5m6wz55antcdt4k4ltmq2u3sj2jm -n myWallet`

##### Arguments
- Use `-c` to specify the CID of the file to renew (required).
- Use `-n` to specify the wallet name to pay for the renewal (required).


#### Unprocessed Pins

Get all database entries with a `validClaim` property of `null` (pins that have not yet been validated):

- `node file-pin.js unprocessed-pins`

##### Arguments
- none


#### Reprocess

Resubmit a pin claim for a CID that has a `validClaim` property of `null`:

- `node file-pin.js reprocess -c bafybeicntn33k36dtrqd6ssf45duek5m6wz55antcdt4k4ltmq2u3sj2jm`

##### Arguments
- Use `-c` to specify the CID of the file to reprocess (required).


### Testing Commands

#### Consumer Test

Test each consumer service listed at `consumers.psfoundation.info`. Checks that each service is connected to the proper BCH and IPFS file service, can retrieve UTXOs, and shows that a specific file is pinned.

- `node file-pin.js consumer-test`

##### Arguments
- none
